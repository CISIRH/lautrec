Déclaration d'accessibilité

Le Centre Interministériel de Services Informatiques relatifs aux Ressources Humaines (CISIRH) s’engage à rendre son service accessible, conformément à l’article 47 de la loi n° 2005-102 du 11 février 2005.

Cette déclaration d’accessibilité s’applique à l'espace DSN.

État de conformité

L'espace DSN est partiellement conforme avec le RGAA 4.1. Partiellement conforme veut dire que certaines sections du contenu ne sont pas entièrement conformes aux standards d'accessibilités.

Résultats des tests

L’audit de conformité réalisé en interne, par la société Sopra HR Software, révèle que 51,06 % des critères RGAA sont respectés.

Contenus non accessibles

Non conformité

Malgré nos efforts, certains contenus sont inaccessibles. Vous trouverez ci-dessous une liste des limitations connues :

  1. Des images porteuses d'information n'ont pas de description détaillée: graphiques ;
  2. Certains textes sont trop peu contrastés avec leur arrière-plan : que ce soit en version normal ou contraste élevé, certains contenus n'atteignent pas le seuil minimum ;
  3. Certains composants d'interface sont trop peu contrastés avec leur arrière-plan ;
  4. Les titres ne sont pas associés aux tableaux: tableaux de syntèses avec filtres intégrés ;
  5. Des tableaux de données ont des en-têtes non déclarés ou non associés avec leurs cellules ;
  6. Des liens ne sont pas suffisamment explicites : dans les tableaux, les liens ont tous le même intitulé qui est restitué "Voir la documentation" ;
  7. Des scripts présents ne sont pas compatibles avec les technologies d'assistance : les boutons du tree view ont un nom accessible non pertinent, de même pour les boutons dans le calendrier... ;
  8. Des scripts ne sont pas accessibles au clavier ou tout autre dispositif de pointage : quand on navigue au clavier, les boutons du tree view ne proposent pas les mêmes actions qu'à la souris (pas de changement de contenu possible au clavier) ;
  9. Les utilisateurs ne sont pas systématiquement prévenus d'un changement de contexte initié par un script : après avoir choisi une option pour filtrer les organisme affichés, le focus clavier se redirige hors de la page sans que l'utilisateur en soit averti ;
  10. Des messages de statut ne sont pas restitués par les technologies d'assistance : sur la page espace fiscal, les messages "10 déclarations extraites et non transmises en M2M." manquent d'attribut ;
  11. Des balises sont utilisées uniquement à des fins de présentation: du contenu textuel n'est pas structuré ;
  12. Le contenu n'est pas toujours correctement structuré par des titres : Aucun titre n'est présent sur la majorité des pages ;
  13. La structure du document n'est pas toujours cohérente : Le contenu principal n'est pas structuré dans la balise adéquate ;
  14. Certaines listes ne sont pas correctement structurées : les liens de la navigation principale ;
  15. Les feuilles de style ne sont pas systématiquement utilisées : présence d'attributs ne devant pas être intégrés dans le code source généré de la page (color, height) ;
  16. Du contenu n'est plus compréhensible lorsque les feuilles de styles sont désactivées : du contenu non visible sur l'interface, apparaît lorsque le CSS est désactivé ;
  17. Certains contenus ne sont plus lisibles lorsque l'utilisateur applique un zoom de 200% : du contenu disparaît (sous-menu du tree view) ;
  18. Des déclarations CSS de couleurs de fond et de police ne sont pas correctement utilisées : Les propriétés CSS sont absentes (couleur de texte et couleur d'arrière-plan) ;
  19. L'espacement du texte cause une perte de contenu ou de fonctionnalité : Sur la page espace fiscal, le pourcentage dans le tableau ;
  20. Le contrôle de saisie n'est pas correctement utilisé: sur la page de connexion, le message d'erreur n'est pas structuré et les champs obligatoires manque d'attribut ;
  21. Des champs manquent d'une déduction de saisie pour faciliter le remplissage automatique : sur la page d'accueil d'une déclaration – Section de recherche individus, les champs nom et prénom ;
  22. L'utilisateur n'a pas le contrôle de chaque limite de temps modifiant le contenu.

L'ensemble de ces points de non conformité sera intégré aux plans d'action annuels du schéma pluriannuel de mise en accessibilité du CISIRH.

Outils utilisés pour évaluer l'accessibilité

  1. Module ANDI (Accessible Name and Description Inspector) ;
  2. Colour Contrast Analyser ;
  3. Extension Web Developer ;
  4. Outils pour développeurs intégrés au navigateur Firefox ;
  5. Validateur HTML du W3C.

Dérogations pour charge disproportionnée

Le CISIRH n'invoque la dérogation temporaire pour charge disproportionnée sur aucune section, aucun contenu et aucune fonction de l'application.

Établissement de cette déclaration d'accessibilité

Cette déclaration a été établie le 6 février 2025.

Amélioration et contact

Si vous n’arrivez pas à accéder à un contenu ou à un service en raison d'un défaut d'adaptation de l'application à votre handicap, vous pouvez contacter le responsable accessibilité de l'espace DSN pour être orienté vers une alternative accessible ou obtenir le contenu sous une autre forme.

  • E-mail : aurelien.louche@finances.gouv.fr
  • Adresse : 41 boulevard Vincent Auriol, 75013 Paris

Important: merci de n’utiliser ces coordonnées qu’en cas de défaut d'adaptation de l'application à votre handicap. Si vous avez besoin d’assistance sur un autre sujet, vous pouvez contacter votre assistance utilisateur locale.

Voie de recours

Cette procédure est à utiliser dans le cas suivant : étant en situation de handicap, vous avez signalé au responsable accessibilité de l'application un défaut d’accessibilité qui vous empêche d’accéder à un contenu ou à un des services de l'application et vous n’avez pas obtenu de réponse dans un délai satisfaisant.

Vous pouvez :

  • Écrire un message au Défenseur des droits
  • Contacter le délégué du Défenseur des droits dans votre région
  • Envoyer un courrier par la poste (gratuit, ne pas mettre de timbre) :
